> 文档版本：v2.0  
> 最后更新：2026-04-05  
> 所属项目：二次元美图开发引擎

## 一、模块概述

"图片修补"是二次元美图开发引擎的核心功能模块之一，面向二次元图片的局部缺陷修复场景，提供以任务为单位的图片修补工作流。用户可以上传待修补的主图、填写修补描述（Prompt）、选择参考图，由 AI 引擎生成修补结果，并支持对结果进行预览、下载和二次修补。

### 设计目标

- **轻量前台**：专注于任务管理与结果展示，界面简洁不冗余
- **风格统一**：延续首页的 pastel 二次元气质，保持整体视觉一致性
- **流程完整**：从任务创建 → 图片上传 → Prompt 编写 → 结果查看 → 二次修补，形成完整闭环
- **易用友好**：拖拽上传、模板一键填入、键盘快捷键等细节提升操作效率

## 二、页面结构

模块采用**三栏布局**，整体包裹在一张磨砂玻璃卡片中，与首页风格保持一致。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏（返回按钮 + 模块标题 + 副标语）                    │
├──────────────┬──────────────────────────┬───────────────────┤
│              │                          │                   │
│  左侧         │  中间                    │  右侧             │
│  任务列表区   │  任务编辑区               │  结果展示区        │
│  (224px)     │  (flex-1, 自适应)         │  (320px)          │
│              │                          │                   │
└──────────────┴──────────────────────────┴───────────────────┘
```

### 2.1 顶部导航栏

| 元素 | 说明 |
|------|------|
| 返回按钮 | 圆形图标按钮，点击返回首页（`/`） |
| 模块图标 | 渐变粉色圆角方块，内含橡皮擦图标 |
| 模块标题 | "图片修补"，ZCOOL 快乐体，粉玫红渐变色 |
| 副标语 | "智能修复，让每一张图都完美无瑕"，中等屏幕以上显示，半透明淡粉色 |

### 2.2 左侧 — 任务列表区

- **标题栏**：显示"修补任务"标题 + 右侧"新建"渐变按钮
- **任务项**：每项显示状态圆点、任务名称、状态标签；选中项左侧有粉色竖线高亮
- **删除**：鼠标悬停时显示删除图标，点击删除（不影响其他任务）
- **底部**：显示当前任务总数

**任务状态定义：**

| 状态值 | 显示文字 | 颜色 |
|--------|----------|------|
| `pending` | 未开始 | 灰色 |
| `processing` | 处理中 | 琥珀色（脉冲动画） |
| `completed` | 已完成 | 玫红色 |
| `failed` | 失败 | 红色 |

### 2.3 中间 — 任务编辑区

编辑区从上到下依次包含五个功能区块：

#### ① 待修补主图上传区
- 支持**点击上传**和**拖拽上传**两种方式
- 仅允许上传 1 张图片（JPG / PNG / WEBP）
- 上传后显示图片预览（`aspect-video`，`object-contain`）
- 悬停预览图时显示"更换图片"按钮，可重新上传

#### ② 修补 Prompt 输入区
- 多行文本框（4 行），支持自由输入修补描述
- 右上角 **「新建模板」**：打开大尺寸表单弹窗，创建自定义 Prompt 模板（与后端 `/api/repair/templates` 对接）
- 右上角 **「选用模板」**：展开模板列表面板；列表数据来自接口，加载中显示占位提示

#### ③ Prompt 模板列表面板
- 展示当前环境下通过接口保存的全部模板，点击条目主体将 **模板内容（text）** 填入 Prompt 输入框并收起面板
- **不再提供** 系统预置默认模板；模板均由用户通过「新建模板」创建（后端 `is_builtin=false`）
- 每条模板右侧提供铅笔（修改）、垃圾桶（删除）；若接口返回 `is_builtin=true` 的历史数据（极少见），则不显示编辑/删除，与后端「内置不可改删」策略一致
- 删除前在同一行展开 **确认 / 取消** 二次确认
- **空列表**：当接口返回 0 条模板时，展示引导文案，提示用户通过「新建模板」创建

#### ④ 参考图上传区
- 最多上传 5 张参考图，以 64×64 缩略图展示
- 每张缩略图悬停时显示删除按钮（右上角红色圆形）
- 未达上限时显示虚线"+"添加按钮

#### ⑤ 输出数量选择区
- 三个选项：**1 张 / 2 张 / 4 张**
- 选中项使用粉玫红渐变填充，未选中为浅玫色

#### 提交按钮
- 文字"开始修补"，ZCOOL 快乐体，粉玫红渐变背景
- 未满足条件（无主图或无 Prompt）时置灰禁用
- 处理中时显示旋转加载图标 + "修补中…"文字

### 2.4 右侧 — 结果展示区

根据当前状态显示三种视图：

| 状态 | 显示内容 |
|------|----------|
| 空状态 | 虚线边框图标 + 引导文字 + 装饰圆点 |
| 处理中 | 旋转加载动画 + 骨架屏占位格 |
| 已完成 | 结果图片网格 |

**结果图片网格布局：**

| 输出数量 | 布局 |
|----------|------|
| 1 张 | 单列大图 |
| 2 张 | 2 列双图 |
| 4 张 | 2×2 网格 |

每张结果图悬停时显示：
- **保存**按钮：直接下载当前图片
- **继续修补**按钮：将此图设为下一次修补的主图输入

## 三、Prompt 模板管理功能（v2.0）

### 3.1 新建 / 编辑弹窗（`TemplateModal`）

- **触发**：Prompt 区域右上角「新建模板」；或模板列表中点击铅笔图标（预填当前标题、描述、内容；`is_builtin=true` 的项不显示铅笔）
- **布局**：全屏半透明遮罩 + 居中卡片，**最大高度 90vh**，标题栏与底栏固定，**中间表单区域可滚动**
- **字段与校验**

| 字段 | 规则 |
|------|------|
| 模板标题 | 必填，最多 40 字（提交至后端时字段名为 `label`，后端上限为 100，前端收紧为 40） |
| 模板描述 | 选填，最多 100 字；随创建/更新请求写入后端 `description` 字段。`enrich` 在接口未返回描述时可用 `localStorage`（键 `aetherframe_repair_tpl_desc_v1`）兜底旧数据 |
| 模板内容 | 必填，最多 5000 字，对应后端字段 `text`；输入框下方 **实时字数统计** |

- **交互**：`ESC` 关闭；点击遮罩（非卡片区域）关闭；「取消」关闭；「创建模板」/「保存修改」通过校验后调用创建或更新接口并刷新列表

### 3.2 与任务切换的联动

- 切换当前修补任务时，自动收起模板面板、关闭模板弹窗、取消进行中的删除确认，避免跨任务误操作（由 `TaskEditor` 的 `taskId` 属性驱动）

### 3.3 接口与类型说明

- 列表接口返回的 `data` 为 `{ templates, total }`，前端在 `repairApi.getTemplates` 中解包为 `templates` 数组
- `PromptTemplate`（`src/types/repair.ts`）与后端一致：`label`、`text`、`is_builtin`、`sort_order`、`created_at` 等；前端展示用扩展类型见 `src/mocks/repairTasks.ts`（增加 `description`）

## 四、大图预览弹窗

点击任意结果图片，弹出全屏预览弹窗（`ImagePreviewModal` 组件）。

### 弹窗结构

```
┌─────────────────────────────────────────┐
│  ● ● ●（图片指示点）          [✕ 关闭]  │
│                                         │
│  [←]   [ 大图预览区域 ]   [→]           │
│                                         │
│              1 / 2                      │
│                                         │
│    [下载图片]      [继续修补]            │
│                                         │
│      按 ← → 切换图片 · ESC 关闭         │
└─────────────────────────────────────────┘
```

### 交互说明

| 操作 | 行为 |
|------|------|
| 点击背景遮罩 | 关闭弹窗 |
| 点击 ✕ 按钮 | 关闭弹窗 |
| 键盘 `ESC` | 关闭弹窗 |
| 键盘 `←` / `→` | 切换上一张 / 下一张 |
| 点击左右箭头 | 切换图片 |
| 点击指示点 | 跳转到对应图片 |
| 点击"下载图片" | 触发浏览器下载，文件名为 `repair-result-N.png` |
| 点击"继续修补" | 将当前图设为主图，重置任务状态，关闭弹窗 |

## 五、核心交互流程

### 4.1 标准修补流程

```
新建任务
  ↓
上传主图（点击 / 拖拽）
  ↓
填写 Prompt（手动输入 或 选用模板）
  ↓
（可选）上传参考图
  ↓
选择输出数量（1 / 2 / 4）
  ↓
点击"开始修补"
  ↓
任务状态 → processing（右侧显示骨架屏）
  ↓
（模拟 3 秒后）任务状态 → completed
  ↓
右侧展示结果图片网格
```

### 4.2 继续修补流程

```
查看结果图片
  ↓
悬停图片 → 点击"继续修补"
  或
点击图片 → 打开预览弹窗 → 点击"继续修补"
  ↓
当前任务主图 → 替换为所选结果图
任务状态 → pending，结果清空
  ↓
修改 Prompt（可选）
  ↓
再次点击"开始修补"
```

## 七、数据结构

### RepairTask

```typescript
interface RepairTask {
  id: string;              // 任务唯一 ID
  name: string;            // 任务名称
  status: TaskStatus;      // 任务状态
  createdAt: string;       // 创建日期（YYYY-MM-DD）
  mainImage: string;       // 主图 base64 或 URL
  prompt: string;          // 修补 Prompt
  referenceImages: string[]; // 参考图列表（最多 5 张）
  outputCount: 1 | 2 | 4; // 输出数量
  results: string[];       // 结果图片 URL 列表
}

type TaskStatus = "pending" | "processing" | "completed" | "failed";
```

### EditorState（TaskEditor 内部状态）

```typescript
interface EditorState {
  mainImage: string;
  prompt: string;
  referenceImages: string[];
  outputCount: 1 | 2 | 4;
}
```

### PromptTemplate（API，`src/types/repair.ts`）

与后端 `PromptTemplateResponse` 一致：

```typescript
interface PromptTemplate {
  id: string;
  label: string;
  description: string;   // 列表简短说明
  text: string;           // 模板正文，填入修补 Prompt
  is_builtin: boolean;
  sort_order: number;
  created_at: string;
}
```

### PromptTemplate（展示用，`src/mocks/repairTasks.ts`）

在 API 类型基础上增加仅前端使用的 `description`（列表副标题），由 `enrichPromptTemplate(s)` 生成。

## 十、当前限制与后续规划

### 当前限制

- **无持久化**：任务数据仅保存在内存中，刷新页面后丢失
- **模拟处理**：AI 修补逻辑为前端模拟（3 秒延迟），尚未接入真实 API
- **无任务重命名**：任务名称创建后不可修改
- **无对比查看**：暂不支持原图与结果图的并排对比

### 后续规划

| 优先级 | 功能 |
|--------|------|
| 高 | 接入真实 AI 修补 API（Supabase Edge Function） |
| 高 | IndexedDB 持久化任务数据，刷新不丢失 |
| 中 | 任务名称双击重命名 |
| 中 | 原图 / 结果图左右对比查看 |
| 低 | 任务导出（打包下载所有结果图） |
| 低 | 修补历史版本回溯 |
