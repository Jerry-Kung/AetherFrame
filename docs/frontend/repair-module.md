## 一、模块概述

"图片修补"是二次元美图开发引擎的核心功能模块之一，面向二次元图片的局部缺陷修复场景，提供以任务为单位的图片修补工作流。用户可以上传待修补的主图、填写修补描述（Prompt）、选择参考图，由 AI 引擎生成修补结果，并支持对结果进行预览、下载和二次修补。

### 设计目标

- **轻量前台**：专注于任务管理与结果展示，界面简洁不冗余
- **风格统一**：延续首页的 pastel 二次元气质，保持整体视觉一致性
- **流程完整**：从任务创建 → 图片上传 → Prompt 编写 → 结果查看 → 二次修补，形成完整闭环
- **易用友好**：拖拽上传、模板一键填入、键盘快捷键等细节提升操作效率

## 二、页面结构

模块采用**三栏布局**，整体包裹在一张磨砂玻璃卡片中，与首页风格保持一致。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏（返回按钮 + 模块标题 + 副标语）                    │
├──────────────┬──────────────────────────┬───────────────────┤
│              │                          │                   │
│  左侧         │  中间                    │  右侧             │
│  任务列表区   │  任务编辑区               │  结果展示区        │
│  (224px)     │  (flex-1, 自适应)         │  (320px)          │
│              │                          │                   │
└──────────────┴──────────────────────────┴───────────────────┘
```

### 2.1 顶部导航栏

| 元素 | 说明 |
|------|------|
| 返回按钮 | 圆形图标按钮，点击返回首页（`/`） |
| 模块图标 | 渐变粉色圆角方块，内含橡皮擦图标 |
| 模块标题 | "图片修补"，ZCOOL 快乐体，粉玫红渐变色 |
| 副标语 | "智能修复，让每一张图都完美无瑕"，中等屏幕以上显示，半透明淡粉色 |

### 2.2 左侧 — 任务列表区

- **标题栏**：显示"修补任务"标题 + 右侧"新建"渐变按钮
- **任务项**：每项显示状态圆点、任务名称、状态标签；选中项左侧有粉色竖线高亮
- **删除**：鼠标悬停时显示删除图标，点击删除（不影响其他任务）
- **底部**：显示当前任务总数

**任务状态定义：**

| 状态值 | 显示文字 | 颜色 |
|--------|----------|------|
| `pending` | 未开始 | 灰色 |
| `processing` | 处理中 | 琥珀色（脉冲动画） |
| `completed` | 已完成 | 玫红色 |
| `failed` | 失败 | 红色 |

### 2.3 中间 — 任务编辑区

编辑区从上到下依次包含五个功能区块：

#### ① 待修补主图上传区
- 支持**点击上传**和**拖拽上传**两种方式
- 仅允许上传 1 张图片（JPG / PNG / WEBP）
- 上传后显示图片预览（`aspect-video`，`object-contain`）
- 悬停预览图时显示"更换图片"按钮，可重新上传

#### ② 修补 Prompt 输入区
- 多行文本框（4 行），支持自由输入修补描述
- 右上角"选用模板"按钮，点击展开模板面板

#### ③ Prompt 模板选择区
- 内置 6 个预置模板（见下方模板列表）
- 点击模板项自动填入 Prompt 输入框，面板自动收起

**内置模板：**

| 模板名称 | 适用场景 |
|----------|----------|
| 皮肤瑕疵修补 | 修复人物皮肤瑕疵，保持动漫风格 |
| 水印去除 | 去除水印、文字标记及 logo |
| 背景噪点修复 | 修复背景噪点、模糊和色差 |
| 角色轮廓补全 | 补全残缺或被遮挡的角色轮廓 |
| 服装细节修复 | 修复衣物皱褶、配饰残损等细节 |
| 眼睛高光补绘 | 为角色眼睛添加或修复高光点 |

#### ④ 参考图上传区
- 最多上传 5 张参考图，以 64×64 缩略图展示
- 每张缩略图悬停时显示删除按钮（右上角红色圆形）
- 未达上限时显示虚线"+"添加按钮

#### ⑤ 输出数量选择区
- 三个选项：**1 张 / 2 张 / 4 张**
- 选中项使用粉玫红渐变填充，未选中为浅玫色

#### 提交按钮
- 文字"开始修补"，ZCOOL 快乐体，粉玫红渐变背景
- 未满足条件（无主图或无 Prompt）时置灰禁用
- 处理中时显示旋转加载图标 + "修补中…"文字

### 2.4 右侧 — 结果展示区

根据当前状态显示三种视图：

| 状态 | 显示内容 |
|------|----------|
| 空状态 | 虚线边框图标 + 引导文字 + 装饰圆点 |
| 处理中 | 旋转加载动画 + 骨架屏占位格 |
| 已完成 | 结果图片网格 |

**结果图片网格布局：**

| 输出数量 | 布局 |
|----------|------|
| 1 张 | 单列大图 |
| 2 张 | 2 列双图 |
| 4 张 | 2×2 网格 |

每张结果图悬停时显示：
- **保存**按钮：直接下载当前图片
- **继续修补**按钮：将此图设为下一次修补的主图输入

## 三、大图预览弹窗

点击任意结果图片，弹出全屏预览弹窗（`ImagePreviewModal` 组件）。

### 弹窗结构

```
┌─────────────────────────────────────────┐
│  ● ● ●（图片指示点）          [✕ 关闭]  │
│                                         │
│  [←]   [ 大图预览区域 ]   [→]           │
│                                         │
│              1 / 2                      │
│                                         │
│    [下载图片]      [继续修补]            │
│                                         │
│      按 ← → 切换图片 · ESC 关闭         │
└─────────────────────────────────────────┘
```

### 交互说明

| 操作 | 行为 |
|------|------|
| 点击背景遮罩 | 关闭弹窗 |
| 点击 ✕ 按钮 | 关闭弹窗 |
| 键盘 `ESC` | 关闭弹窗 |
| 键盘 `←` / `→` | 切换上一张 / 下一张 |
| 点击左右箭头 | 切换图片 |
| 点击指示点 | 跳转到对应图片 |
| 点击"下载图片" | 触发浏览器下载，文件名为 `repair-result-N.png` |
| 点击"继续修补" | 将当前图设为主图，重置任务状态，关闭弹窗 |

## 四、核心交互流程

### 4.1 标准修补流程

```
新建任务
  ↓
上传主图（点击 / 拖拽）
  ↓
填写 Prompt（手动输入 或 选用模板）
  ↓
（可选）上传参考图
  ↓
选择输出数量（1 / 2 / 4）
  ↓
点击"开始修补"
  ↓
任务状态 → processing（右侧显示骨架屏）
  ↓
（模拟 3 秒后）任务状态 → completed
  ↓
右侧展示结果图片网格
```

### 4.2 继续修补流程

```
查看结果图片
  ↓
悬停图片 → 点击"继续修补"
  或
点击图片 → 打开预览弹窗 → 点击"继续修补"
  ↓
当前任务主图 → 替换为所选结果图
任务状态 → pending，结果清空
  ↓
修改 Prompt（可选）
  ↓
再次点击"开始修补"
```


## 六、数据结构

### RepairTask

```typescript
interface RepairTask {
  id: string;              // 任务唯一 ID
  name: string;            // 任务名称
  status: TaskStatus;      // 任务状态
  createdAt: string;       // 创建日期（YYYY-MM-DD）
  mainImage: string;       // 主图 base64 或 URL
  prompt: string;          // 修补 Prompt
  referenceImages: string[]; // 参考图列表（最多 5 张）
  outputCount: 1 | 2 | 4; // 输出数量
  results: string[];       // 结果图片 URL 列表
}

type TaskStatus = "pending" | "processing" | "completed" | "failed";
```

### EditorState（TaskEditor 内部状态）

```typescript
interface EditorState {
  mainImage: string;
  prompt: string;
  referenceImages: string[];
  outputCount: 1 | 2 | 4;
}
```

## 七、视觉设计规范

### 色彩体系

| 用途 | 色值 |
|------|------|
| 主渐变（按钮、标题） | `#f472b6 → #ec4899 → #db2777` |
| 背景底色 | `linear-gradient(145deg, #fff5f7, #fffaf5, #fef2f8, #fff8f0)` |
| 卡片背景 | `rgba(255,255,255,0.55)` + `backdrop-filter: blur(20px)` |
| 分割线 | `rgba(254,205,211,0.6)` |
| 辅助文字 | `rgba(244,114,182,0.4)` |

### 字体

| 用途 | 字体 |
|------|------|
| 标题、按钮 | ZCOOL KuaiLe（ZCOOL 快乐体，Google Fonts） |

### 圆角规范

| 元素 | 圆角 |
|------|------|
| 主卡片容器 | `rounded-3xl`（24px） |
| 图片预览、结果图 | `rounded-2xl`（16px） |
| 按钮（普通） | `rounded-xl`（12px） |
| 胶囊按钮、标签 | `rounded-full` |

### 动效

| 动效 | 应用场景 |
|------|----------|
| `floatUp`（上下浮动） | 背景装饰气泡 |

## 九、当前限制与后续规划

### 当前限制

- **无持久化**：任务数据仅保存在内存中，刷新页面后丢失
- **模拟处理**：AI 修补逻辑为前端模拟（3 秒延迟），尚未接入真实 API
- **无任务重命名**：任务名称创建后不可修改
- **无对比查看**：暂不支持原图与结果图的并排对比

### 后续规划

| 优先级 | 功能 |
|--------|------|
| 高 | 接入真实 AI 修补 API（Supabase Edge Function） |
| 高 | IndexedDB 持久化任务数据，刷新不丢失 |
| 中 | 任务名称双击重命名 |
| 中 | 原图 / 结果图左右对比查看 |
| 低 | 任务导出（打包下载所有结果图） |
| 低 | 修补历史版本回溯 |
