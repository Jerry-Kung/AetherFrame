# 素材加工模块 — 设计说明文档

> 文档版本：v1.0  
> 最后更新：2026-04-07  
> 所属项目：二次元美图开发引擎

---

## 一、模块概述

"素材加工"是二次元美图开发引擎的核心功能模块之一，面向以角色为中心的轻量素材整理与加工场景。用户可以创建/管理角色、上传设定文本与参考图、查看加工进度、浏览生成的标准参考照与结构化角色小档案。

### 设计目标

- **角色为核心**：所有功能围绕角色展开，提供角色列表、档案、编辑与加工入口
- **轻量前台**：专注于角色资料管理与加工流程展示，界面简洁不冗余
- **风格统一**：延续首页的 pastel 二次元气质，保持整体视觉一致性
- **流程完整**：从角色创建 → 原始资料整理 → 加工任务 → 正式内容展示，形成完整闭环
- **易用友好**：标签分类、悬停预览、大图浏览等细节提升操作效率

## 二、页面结构

模块采用**三栏布局**，整体包裹在一张磨砂玻璃卡片中，与首页、修补页风格保持一致。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏（返回按钮 + 模块标题 + 副标语）                    │
├──────────────┬──────────────────────────┬───────────────────┤
│              │                          │                   │
│  左侧         │  中间                    │  右侧             │
│  角色列表区   │  三页签主区域            │  角色档案侧边栏   │
│  (216px)     │  (flex-1, 自适应)         │  (288px)          │
│              │                          │                   │
└──────────────┴──────────────────────────┴───────────────────┘
```

### 2.1 顶部导航栏

| 元素 | 说明 |
|------|------|
| 返回按钮 | 圆形图标按钮，点击返回首页（`/`） |
| 模块图标 | 渐变粉色圆角方块，内含剪刀图标 |
| 模块标题 | "素材加工"，ZCOOL 快乐体，粉玫红渐变色 |
| 副标语 | "以角色为中心，整理设定与参考，生成标准档案"，中等屏幕以上显示，半透明淡粉色 |

### 2.2 左侧 — 角色列表区

- **标题栏**：显示"你的小女友们"标题 + 右侧"新建"渐变按钮
- **角色项**：每项显示缩略图、名称、状态圆点、状态标签、更新时间；选中项左侧有粉色竖线高亮
- **删除**：鼠标悬停时显示删除图标，点击删除（不影响其他角色）
- **空状态**：无角色时显示提示文案与装饰图标

**角色状态定义：**

| 状态值 | 显示文字 | 颜色 |
|--------|----------|------|
| `idle` | 还没开始 | 灰色 |
| `draft` | 资料待补充 | 琥珀色 |
| `processing` | 加工进行中 | 玫红色（脉冲动画） |
| `done` | 已整理完成 | 绿色 |

### 2.3 中间 — 三页签主区域

主区域从上到下依次包含：

#### ① 面包屑导航
显示当前选中角色名，提供上下文感知。

#### ② 主页签切换
三个页签：原始资料 / 加工任务 / 正式内容，选中项使用粉玫红渐变样式。

---

#### 页签一：原始资料

**设定说明区域：**
- 多行文本框，支持自由编辑角色设定
- 右上角「上传 .txt / .md」：导入文本文件（导入内容会整段替换当前编辑区）
- 文本文件编码：UTF-8

**原始参考图区域：**
- 多图上传，支持 JPG / PNG / WEBP 格式
- 标签预设：立绘 / 三视图 / 表情 / 服装 / 场景 / 其他（默认选中「其他」）
- 标签 chips 过滤：点击标签只显示对应标签的图片
- 图片网格展示：响应式网格，悬停时显示大图预览浮层
- 删除按钮：悬停时显示，点击删除单张图片
- 点击图片：打开全屏大图预览弹窗

**资料就绪横幅：**
- 判定条件：`settingText.trim().length > 0 && rawImages.length > 0`
- 就绪：绿色提示栏，显示"资料已就绪"
- 未就绪：琥珀色提示栏，显示"资料尚未就绪"及引导文案

---

#### 页签二：加工任务

**子任务切换栏：**
- 两个子页签：拍摄标准照 / 角色小档案
- 选中项使用粉玫红渐变样式

**主操作区占位：**
- 显示当前子任务标题与任务说明
- 大面积虚线边框区域占位，标注"后续流程展开位置"

---

#### 页签三：正式内容

**标准参考照展示：**
- 三格固定布局，显示标准参考照
- 有图可点：点击打开全屏大图预览
- 无图占位：显示虚线边框与"待生成"提示

**角色小档案卡片：**
- 结构化展示：姓名 / 年龄 / 身高 / 性格 / 能力 / 外观
- 定义列表式布局，标题与内容清晰区分

---

### 2.4 右侧 — 角色档案侧边栏

侧边栏从上到下依次包含：

#### ① 角色卡片
- 角色代表图（大头像）
- 角色名称 + 状态圆点
- 当前状态文字标签

#### ② 资料小结
- 设定说明摘要（最多 72 字符），超出显示省略号

#### ③ 统计数据
- 参考图数量
- 标准照完成度（如 2 / 3）

#### ④ 快捷操作按钮
- **编辑角色信息**：打开简易 Modal（编辑列表名称与档案显示名）
- **开始加工任务**：切到「加工任务」页签并显示 toast 提示
- **导出角色档案**：下载当前角色 JSON（本地预览用）

## 三、核心组件说明

### 3.1 大图预览弹窗（`ImagePreviewModal`）

**复用组件**：直接复用修补模块的 `ImagePreviewModal`

**配置差异**：
- `showContinueRepair={false}`：不显示"继续修补"按钮
- `imageAltPrefix`：根据来源设置为"参考图"或"标准参考照"
- `onDownload`：支持直接下载，文件名格式为 `image-{N}.png`

**交互说明**（与修补页一致）：

| 操作 | 行为 |
|------|------|
| 点击背景遮罩 | 关闭弹窗 |
| 点击 ✕ 按钮 | 关闭弹窗 |
| 键盘 `ESC` | 关闭弹窗 |
| 键盘 `←` / `→` | 切换上一张 / 下一张 |
| 点击左右箭头 | 切换图片 |
| 点击指示点 | 跳转到对应图片 |
| 点击"下载图片" | 触发浏览器下载 |

## 五、数据结构

### CharaStatus（角色加工四态）

```typescript
type CharaStatus = "idle" | "draft" | "processing" | "done";
```

### CharaRawImage（原始参考图）

```typescript
interface CharaRawImage {
  id: string;
  url: string;
  tags: string[];
}
```

### CharaBio（结构化小档案）

```typescript
interface CharaBio {
  displayName: string;
  age: string;
  height: string;
  personality: string;
  ability: string;
  appearance: string;
}
```

### CharaProfile（完整角色档案）

```typescript
interface CharaProfile {
  id: string;
  name: string;
  avatarUrl: string;
  status: CharaStatus;
  updatedAt: string;         // ISO 时间字符串
  settingText: string;       // 原始资料：设定说明
  rawImages: CharaRawImage[]; // 原始参考图
  officialPhotos: [string | null, string | null, string | null]; // 标准参考照（固定三槽）
  bio: CharaBio;             // 结构化小档案
}
```

### 常量配置

- `STATUS_LABEL`：状态值 → 中文文案映射
- `STATUS_STYLE`：状态值 → 样式配置（圆点、文字、标签）
- `RAW_IMAGE_TAG_PRESETS`：标签预设列表

### 标准参考照槽位删除（后端 API）

正式内容页「删除标准照」调用素材模块接口，与前端 [`materialApi.deleteOfficialPhotoSlot`](../../page/src/services/materialApi.ts) 一致。

| 项目 | 说明 |
|------|------|
| 方法 / 路径 | `DELETE` `/api/material/characters/{character_id}/standard-photo/slot/{slot_index}` |
| `slot_index` | 整数 `0`–`4`，顺序为：全身正面、全身侧面、半身正面、半身侧面、脸部特写（与加工任务标准照类型一致） |
| 成功响应 | HTTP `200`，`success: true`，`data` 为完整 `CharacterDetail`（`official_photos` 五元组中对应槽为 `null`） |
| `404` | 角色不存在（`detail`:「角色不存在」） |
| `400` | 槽位索引非法（不在 `0`–`4`） |
| 语义与幂等 | 清空该槽在库中的 URL，并删除槽位 PNG（`standard_photo_slots/{shot_type}.png`）；不删除标准照生成任务记录。槽位已为空时再次删除仍返回成功与最新详情。 |
| 实现顺序 | 服务端先提交数据库清空，再删除磁盘文件，避免 DB 失败时出现「文件已删但 JSON 仍指向旧 URL」的不一致。 |

## 八、开发环境与生产环境配置差异

### Vite `base` 配置

由于项目使用 FastAPI 托管 `app/static`，开发与生产环境的路由 base 需要区分：

| 环境 | `base` | 说明 |
|------|--------|------|
| 开发（`vite dev`） | `/` | React Router 导航到 `/material` 与 `/repair` 时，Vite 能正确做 SPA 回退，避免 404 |
| 生产（`vite build`） | `/static/` | 资源路径为 `/static/assets/...`，与 FastAPI `StaticFiles` 挂载路径一致 |

配置见 [`page/vite.config.ts`](../../page/vite.config.ts)，使用 `defineConfig(({ command }) => { ... })` 动态区分。

## 九、当前限制与后续规划

### 当前限制

- **无持久化**：角色数据仅保存在内存中，刷新页面后丢失
- **Mock 数据**：当前为纯前端 Mock 实现，尚未接入后端 API
- **加工流程占位**："加工任务"页签主操作区为占位，具体流程需后续接入
- **标准照生成**：标准参考照当前为占位图片，无真实生成逻辑

### 后续规划

| 优先级 | 功能 |
|--------|------|
| 高 | 接入后端角色 CRUD API（列表、详情、更新、删除） |
| 高 | 文件上传：将原始参考图与设定上传到后端存储（OSS 或本地） |
| 高 | 加工任务流程：拍摄标准照 / 角色小档案的具体交互与 API 对接 |
| 中 | 标准参考照生成与展示 |
| 中 | 角色档案导出（支持 PDF 等格式） |
| 低 | 角色标签与分类管理 |
| 低 | 角色批量导入与导出 |
