# Feedback 归因层重构：从配置维度关联转向词汇层分析

- 日期：2026-07-16
- 状态：已实现
- 取代：`2026-07-16-feedback-kb-analysis-design.md` 的归因层（§4 关联分析统计方法中
  "配置维度扫描"部分与报告的特征关联章节）。知识库底座、紧迫度排行榜、版本时间线、
  守门阈值等其余部分继续有效。

## 1. 方向修正的依据

首份 v1 报告产出"崩坏 × 绘图配置（画幅/视线/姿势族）"的 RR 关联证据后，用户判定该方向
**绝对错误**：

1. **无因果通路**：画幅这类绘图配置与崩坏模式之间想象不到逻辑因果关系（"早餐吃什么 ×
   今天天气"式的伪关联挖掘）。
2. **不可干预**：即便关联为真，处方空间也在 Prompt 措辞，不在"禁用某画幅"。分析必须活在
   可干预的空间里。
3. **正确方向是词汇层**（用户既有实战经验）：解剖词过密（足弓/脚背/脚趾反复出现）勾引
   模型注意力导致"足部解剖图"；过度文学化描述（"含情脉脉"）触发模型胡编细节崩坏。

同日实测数据进一步约束了设计（60 case 全库验证）：

| 预定义词表假说 | 崩坏组均值 | 对照组均值 |
|---|---|---|
| 脚袜段解剖词次数 × 脚部/夸张 | 3.00 | 3.33 |
| 褶/皱词次数 × 皱褶夸张 | 1.67 | 1.75 |
| 光泽词次数 × 上色感 | 0.50 | 0.50 |

**零区分度的原因是 rulepack 模板已把固定段措辞同质化**（R3 保护措辞本身含足尖/脚趾，
全员≈3 次解剖词）。结论：崩坏诱因藏在模板自由发挥区（场景/姿势描写/表情/角色特有描述/
种子用词）的**开放词汇**里——"预定义词表数词频"发现不了它们，必须 LLM 整段对比精读；
但精读提出的假说必须回到全库做确定性验证，防止被小样本带偏。

## 2. 新归因闭环：LLM 提假说、代码验假说

```
kb_build ──→ report（排行榜定靶）
                 │
                 ▼
   contrast.py ──→ 对比素材包（崩坏组 vs 姿势族配对对照组，完整 Prompt+种子+反馈）
                 │
                 ▼
   对话层精读（Claude 人机对话，不编码、不调 API）──→ 可检测假说（pattern+scope）
                 │ 用户确认后登记
                 ▼
   lexical_hypotheses.yaml ──→ lexicon_verify（全库 RR + 四道守门）──→ report 词汇假说验证表
                 │
                 ▼
   supported 假说 → 人机对话开方 → exp0NN A/B 验证
```

用户拍板的两个决策：
- **精读在对话层执行**：机制只产素材包文件；假说质量有人把关、无 API 依赖、与
  "LLM 开方留对话层"边界一致。跑顺后若要固化成管线另立议题。
- **配置维度关联彻底删除**：`correlate.py` 及其维度扫描测试删除；KB 的 `composition`
  字段保留备查；`pose_family` 标签保留，用途改为素材包对照组的同姿势配对。

## 3. 实现落点

| 变更 | 文件 |
|---|---|
| KB 补存 `full_prompt`（词汇层数据基础，重建幂等 0 LLM） | `experiments/feedback_kb/kb_build.py` |
| 删除配置维度扫描；RR+守门统计核心迁入假说验证 | `correlate.py`（删）→ `lexicon_verify.py`（新） |
| 对比素材包生成（暴露组按 severity 优先、对照组姿势族分层确定性抽样、上限 15/组） | `contrast.py`（新） |
| 词汇假说库（id 不复用；status: candidate/supported/refuted/retired 人工维护） | `experiments/cases/lexical_hypotheses.yaml`（新） |
| 报告 v2：删特征关联章节，加词汇假说验证表；空假说库可空跑 | `report.py` |
| 守门测试迁移 + 假说/素材包/报告 v2 用例 | `tests/experiments/test_feedback_kb.py` |

统计守门语义不变：N_min=8、单种子驱动 ≥60% 标记、零对照标"无对照样本"、
strong/weak/insufficient 三档。多重比较收敛方式升级：检验数 = 登记假说数（预注册性质），
比 Top-K × 维度全扫收敛得多。

## 4. 边界（不做）

- 不把精读步骤编码进管线；不做假说自动生成/自动改 status。
- 不动 rank/versions/守门阈值；不动归档管线。
- seed 词汇不单独建层：seed_text 随素材包进入精读视野即可。
